Professional Self-Assessment

Throughout this program, I transitioned from writing functional assignments to building structured, secure, and maintainable software systems. The capstone forced me to stop thinking in terms of “does it work?” and start thinking in terms of architecture, scalability, validation, security, and long-term maintainability.

Developing this ePortfolio required me to present my work not as coursework, but as documented evidence of technical ability. I now approach development the way industry expects: with intentional design decisions, documented trade-offs, and clear technical communication.

My goal as a software engineer is to build systems that are reliable, secure, and thoughtfully designed, not just functional.

Collaboration and Communication

Engineering is not a solo activity. Throughout the program, I participated in peer reviews, structured feedback cycles, and technical discussions that required me to defend design decisions and evaluate alternative approaches.

In my recorded code review and written enhancement narratives, I practiced explaining complex technical concepts clearly to an academic audience, similar to communicating system behavior to stakeholders or team members.

Outside of this capstone, my experience in the Microsoft Software and Systems Academy (MSSA) reinforced working through development milestones under real constraints, presenting technical decisions clearly, and maintaining consistent documentation and structure.

I learned that clean architecture, naming conventions, and readable code are not preferences, they are collaboration tools.

Data Structures and Algorithms

My coursework strengthened my ability to design solutions using algorithmic reasoning rather than guesswork.

Instead of relying on built-in methods without thought, I evaluated time complexity, ordering, and efficiency. Implementing sorting logic, rolling averages, and trend detection required careful attention to performance and correctness.

I now approach problems by selecting appropriate data structures, analyzing trade-offs, and considering scalability from the start. That mindset is foundational to professional engineering.

Software Engineering and Databases

Across this program, I worked with layered architecture, MVC design patterns, relational schema design, validation frameworks, and database integration.

Refactoring a console application into a structured ASP.NET Core MVC web application reinforced separation of concerns and maintainability. Integrating relational databases required careful schema design, controlled data persistence, and secure user-specific data handling.

I developed hands-on experience with Entity Framework Core, SQLite, structured queries, model-level validation, and controlled data access. These are practical, production-relevant skills, not just academic exercises.

Security Mindset

Security became a recurring theme across multiple courses. I learned to anticipate vulnerabilities such as hardcoded credentials, insufficient validation, injection risks, and weak authentication practices.

By implementing structured validation, secure database access patterns, and authentication mechanisms, I developed a proactive security mindset. Security is not an afterthought in my design process; it is part of the architecture from the start.

How the Artifacts Work Together

The artifacts in this portfolio demonstrate growth across core computer science domains.

The Mortgage Calculator shows my progression in software engineering and database integration, from console-based logic to structured web architecture with persistent storage and validation.

The Weight Tracker Android application demonstrates algorithm-driven problem solving within a mobile environment. By introducing rolling averages, sorting logic, and trend detection, I transformed simple CRUD functionality into meaningful data analysis.

Together, these projects demonstrate structured software design, algorithmic reasoning, database integration, security awareness, and professional communication.

This portfolio represents not just completed coursework, but documented proof of my ability to design, build, analyze, and communicate software solutions aligned with industry expectations.

Enhancement 2: Algorithms and Data Structures

In this enhancement I extended the Android Weight Tracker application beyond simple data storage by introducing structured data handling and algorithmic analysis. Weight entries are now stored as structured objects and processed using algorithmic logic to generate insights such as rolling averages and trend indicators, allowing the application to present calculated insights rather than raw values alone.

Key Improvements

  • Refactored database queries to return structured collections
  • Applied sorting logic to ensure consistent chronological ordering prior to analysis
  • Designed a rolling average algorithm to smooth short-term fluctuations
  • Implemented trend detection logic based on changes in rolling averages
